## 2. P1: 灌水量预测缺少输入功能

### 2.1 问题描述

当前前端只有决策展示功能，缺少数据输入入口。用户无法：
- 上传温室图片
- 输入当日环境数据（温度、湿度、光照等）
- 触发下一天的灌水量预测

### 2.2 期望功能

```
┌─────────────────────────────────────────────────────────────┐
│                    数据输入 → 预测流程                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户输入:                                                   │
│  ├─ 上传温室图片 (今日图像)                                  │
│  ├─ 输入环境数据:                                            │
│  │   ├─ 日均温度 (°C)                                       │
│  │   ├─ 日均湿度 (%)                                        │
│  │   └─ 日均光照 (lux) 或 太阳辐射 (W/m²)                   │
│  └─ 选择预测日期                                             │
│                                                             │
│  系统处理:                                                   │
│  ├─ YOLO 分割 → 获取叶片/花朵/果实指标                       │
│  ├─ 构建 TSMixer 输入特征 (11维)                            │
│  ├─ 获取历史 95 天数据 (冷启动时用填充数据)                  │
│  └─ TSMixer 预测 → 灌水量                                   │
│                                                             │
│  输出展示:                                                   │
│  ├─ 预测灌水量 (L/m²)                                       │
│  ├─ 模型置信度                                               │
│  ├─ 分割后的图像 (带掩码)                                    │
│  └─ YOLO 检测指标                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

前端"视觉生长分析"区域：

### 3.2 相关资源位置

| 资源 | 路径 |
|------|------|
| 图片目录 | `cucumber-irrigation/data/images/` |
| 推理脚本 | `v11_4seg/tuili.py` |
| 模型架构 | 改进的 YOLOv11n-seg |
| 模型权重 | `G:\Wyatt\v11_4seg\runs\segment\jy_data_precoco_delive\exp21\weights\best.pt` |

### 3.3 推理流程 (参考 tuili.py)

```python
# 推理核心逻辑
1. 图像预处理
   - 原始分辨率: 2880×1620
   - 缩放至: 3200×1920

2. 分块推理
   - 分割为 15 个小块 (3行×5列, 每块 640×640)
   - 每块单独推理

3. 结果合并
   - 拼接回 3200×1920
   - 缩放回 2880×1620

4. 可视化
   - 叠加半透明彩色掩码
   - 类别: leaf(红), terminal(黄), flower(绿), fruit(蓝)
```

### 3.4 需要实现的功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 图像展示 | 显示今日/昨日原始图像 | P0 |
| 分割展示 | 显示 YOLO 分割后的图像 (带掩码) | P0 |
| 指标展示 | 显示叶片/花朵/果实的数量和面积 | P0 |
| 对比模式 | 今日 vs 昨日图像对比 | P1 |
| 检测框切换 | 可开关显示检测框/掩码 | P2 |

### 3.5 后端服务需求

需要创建 YOLO 推理服务：
- 加载模型权重
- 接收图像，执行分割推理
- 返回：分割后图像 + 检测指标

## 4. P3: TSMixer 模型未实际调用

### 4.1 问题描述

当前系统没有实际调用 TSMixer 模型进行灌水量预测，只是前端展示占位数据。

### 4.2 TSMixer 调用逻辑 (参考 requirements1.md 2.3节)

```
┌─────────────────────────────────────────────────────────────┐
│                    TSMixer 滚动预测流程                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入规格:                                                   │
│  ├─ 序列长度: 96 步 (天)                                    │
│  ├─ 特征维度: 11 个特征                                     │
│  └─ 输出: 单一灌水量值 (L/m²)                               │
│                                                             │
│  11 个输入特征:                                              │
│  ├─ 1. temperature (日均温度)                               │
│  ├─ 2. humidity (日均湿度)                                  │
│  ├─ 3. light (日均光照)                                     │
│  ├─ 4. leaf Instance Count (叶片数)                         │
│  ├─ 5. leaf average mask (叶片平均掩码)                     │
│  ├─ 6. flower Instance Count (花朵数)                       │
│  ├─ 7. flower Mask Pixel Count (花朵掩码像素)               │
│  ├─ 8. terminal average Mask Pixel Count (顶芽掩码)         │
│  ├─ 9. fruit Mask average (果实平均掩码)                    │
│  ├─ 10. all leaf mask (全叶掩码总面积)                      │
│  └─ 11. Target (历史灌水量)                                 │
│                                                             │
│  预测执行:                                                   │
│  ├─ 输入 → StandardScaler 标准化                           │
│  ├─ 标准化输入 → TSMixer 模型                               │
│  └─ 模型输出 → 逆标准化 → 灌水量预测值                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 冷启动数据填充
